

## Add Google Login

### Summary
Add a Google sign-in button to the Auth page. The project already has `signInWithGoogle` in AuthContext, but it uses `supabase.auth.signInWithOAuth` directly. Per Lovable Cloud requirements, this must use `lovable.auth.signInWithOAuth` instead. The Configure Social Auth tool will generate the required module.

### Steps

1. **Configure Social Auth** — Use the Configure Social Login tool to generate the `src/integrations/lovable/` module with the `@lovable.dev/cloud-auth-js` package. This enables managed Google OAuth with no extra credentials needed.

2. **Update AuthContext** — Change `signInWithGoogle` to use `lovable.auth.signInWithOAuth("google", ...)` instead of `supabase.auth.signInWithOAuth`. Update redirect to `window.location.origin`.

3. **Add Google button to Auth page** — Add a "Sign in with Google" button with Google icon on the Auth page (`src/pages/Auth.tsx`):
   - Place it below the email/password form with a divider ("or continue with")
   - For login mode: show the button directly
   - For signup mode: show the button below the form (note: Google OAuth users will need role assignment handled separately since role selection happens before form submit)

4. **Handle OAuth callback role assignment** — For new Google sign-in users who don't have a role yet, redirect them to a role selection step or default to 'parent' role.

### Technical Details

- **Files modified**: `src/contexts/AuthContext.tsx`, `src/pages/Auth.tsx`
- **Files generated**: `src/integrations/lovable/` (auto-generated by tool)
- **No database changes needed** — the existing `user_roles` and `profiles` tables + `handle_new_user` trigger will handle new OAuth users
